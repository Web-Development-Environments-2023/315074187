"Please supply us with three differences between Web 1.0, 2.0 & 3.0."
Easy question. The answers are:
1. Web 1.0 was extremely static, while the later iterations were significantly more dynamic and interactive!
2. Web 1.0 was mainly based on HTML, while the later iterations were more based on
more advanced technologies (Like API's and AI)
3. When it comes to data collection, 1.0 was extremely scarce. There was no big focus on gathering Data, and companies could collect user data with no issues.
in Web 2.0, due to the increase in social media focus, data collections became a huge issue, and in 3.0 there is a huge focus on privacy.

Oh my, seems like there is another question!
"Please supply us with some use cases on each of the differences!"
... Ok fine.
1. Safety & Privacy: Let's take Web 1.0 and 3.0
The use case will be an online store. In 1.0, the data would be collected and maybe even saved, without much protection.
in 3.0, due to the larger fovus on privacy and security, we will use multiple encryption techniques to make sure the credit history is safe
2. Interactions: Web 1.0 and 2.0
This use case will be Youtube (or any streaming platform)
In 1.0, the focus would be on delivering the main focus, which is streaming videos/music/whatever other content.
In 2.0, user will be able to interact with one another, comment, follow etc.
3. Content creation & consumption: This time, web 2.0 and 3.0
This use case will be any education platform like Coursera, Udemy etc.
In 2.0, Content is available to users, and is limited by what other users (that are lecturers) upload.
In 3.0, the users (students) are able to also help out content created by other users.
Another thing is AI, which will be able to curate content towards users, depending on their previous learnings.
